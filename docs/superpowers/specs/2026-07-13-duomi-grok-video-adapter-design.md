# 多米 Grok 视频模型接入设计

## 1. 目标

为 Infinite Canvas 增加多米 API 的 Grok 视频异步协议适配，并把它纳入多米渠道模板。第一阶段支持：

- `grok-video-1.5` 文生视频。
- `grok-video-1.5` 公网图片 URL 参考图生视频。
- 在同一个多米渠道中同时配置多米图片协议和多米视频协议。

第一阶段不建设图片上传服务，不把浏览器本地 Base64、Blob URL、IndexedDB 图片或局域网资源自动发送给第三方。

参考文档：<https://s.apifox.cn/b924931e-29c0-4127-b025-d68c90285060/api-427255838>

## 2. 现状与兼容差异

Infinite Canvas 当前已有标准 OpenAI 视频、xAI Grok 视频和 Seedance 视频流程。现有 `grok-imagine-video` 适配器不能直接用于多米 `grok-video-1.5`，差异如下：

- 现有 xAI 流程使用 `Bearer` 鉴权；多米文档要求原始 `Authorization: <API Key>`。
- 现有 xAI 查询路径为 `/v1/videos/{id}`；多米查询路径为 `/v1/videos/tasks/{task_id}`。
- 现有 xAI 适配器拒绝参考图；多米通过 `image_urls` 支持公网参考图。
- 现有 xAI 使用 `resolution`；多米使用 `quality: "720p"`。
- 现有解析器不读取多米成功响应的 `data.videos[].url`。

因此不能只把模型名称加入模型列表，也不能按模型名复用现有 xAI 分支。

## 3. 方案选择

采用渠道级独立视频协议字段 `videoApiFormat`，不按模型名猜测供应商，也不在本次工作中重构全部供应商为通用注册表。

备选方案及取舍：

- 按模型名自动分流：改动较少，但同名模型可能属于不同渠道，容易错误路由。
- 重构通用供应商注册表：长期扩展性更强，但会扩大改动面并增加现有视频流程的回归风险。
- 渠道级独立协议：清楚表达请求契约，能与现有 `imageApiFormat` 对称扩展，并保持其他视频流程不变。

## 4. 渠道配置与模板

在 `ModelChannel` 中新增：

```ts
type VideoApiFormat = "standard" | "duomi";

type ModelChannel = {
    // 现有字段
    videoApiFormat: VideoApiFormat;
};
```

字段含义：

- `standard`：沿用当前 OpenAI、xAI 或 Seedance 视频行为。
- `duomi`：使用多米 Grok 视频异步适配器。

配置界面在“图片协议”旁增加“视频协议”选择。多米图片协议和多米视频协议相互独立，用户可以只启用其中一个。

当前配置页只有“新增渠道”，没有渠道模板入口。实现时将新增操作改为包含以下选项的菜单：

- 空白渠道：保持当前新增渠道行为。
- 多米 API：按下述预设创建一个可继续编辑的渠道。

多米渠道模板负责一次性设置：

- 渠道名称为“多米 API”。
- Base URL 为 `https://duomiapi.com`。
- `apiFormat: "openai"`，沿用现有文本、音频和模型读取的基础请求格式。
- API Key 保持为空，由用户输入自己的多米密钥。
- `imageApiFormat: "duomi"`。
- `videoApiFormat: "duomi"`。
- 模型建议包含 `gpt-image-2`、`gemini-2.5-flash-image`、`gemini-3-pro-image-preview`、`gemini-3.1-flash-image-preview` 和 `grok-video-1.5`。

模板只是创建渠道时的便捷预设，不锁定字段。创建后用户仍可修改名称、Base URL、调用格式、图片协议、视频协议和模型列表。

切换到多米视频协议时，将 `grok-video-1.5` 合并到渠道模型建议中，不删除用户手动添加的模型。切回标准视频协议时保留现有模型列表，避免配置丢失。

持久化迁移规则：旧渠道缺少 `videoApiFormat` 时补为 `standard`。迁移不得修改既有 `imageApiFormat`、Base URL、API Key、模型列表或默认模型。

## 5. 组件边界

### 5.1 配置层

配置层负责：

- 定义和归一化 `videoApiFormat`。
- 保存、迁移和解析模型所属渠道的视频协议。
- 在配置界面展示标准视频和多米异步视频选项。
- 合并多米图片与视频建议模型。

### 5.2 纯协议工具

新增可独立测试的多米视频协议工具，负责：

- 创建路径和查询路径构造。
- 原始鉴权头构造。
- 模型、提示词、比例、时长、质量、参考图和 `oversea` 参数映射。
- 创建响应任务 ID 提取。
- 任务状态归一化。
- 服务端错误信息提取。
- 成功响应视频 URL 提取。
- 公网参考图 URL 校验。

纯协议工具不发起网络请求，也不依赖 React 或状态管理。

### 5.3 任务生命周期

新增多米视频任务生命周期模块，负责：

- 创建异步任务。
- 使用 `AbortSignal` 取消创建、轮询请求和轮询等待。
- 轮询任务状态。
- 将成功、失败、超时和异常响应转换成现有视频生成结果结构。

现有 `createVideoGenerationTask` 和 `pollVideoGenerationTask` 只负责根据模型所属渠道的 `videoApiFormat` 分发到标准流程或多米流程。现有 OpenAI、xAI 和 Seedance 分支保持原行为。

## 6. 请求数据流

### 6.1 创建任务

1. 根据所选模型解析对应渠道。
2. 如果渠道 `videoApiFormat !== "duomi"`，继续现有视频流程。
3. 如果为多米视频协议，先校验所有参考图。
4. 请求 `POST /v1/videos/generations`。
5. 使用原始 `Authorization: <API Key>`，不添加 `Bearer`。
6. 从创建响应顶层 `id` 读取任务 ID。
7. 缺少有效任务 ID 时立即返回协议错误，不进入轮询。

请求体为：

```json
{
  "model": "grok-video-1.5",
  "prompt": "画布提示词",
  "aspect_ratio": "16:9",
  "duration": 6,
  "quality": "720p",
  "image_urls": [],
  "oversea": false
}
```

参数映射：

- `model` 使用渠道模型名，不携带渠道编码前缀。
- `prompt` 使用画布当前视频提示词。
- `aspect_ratio` 复用现有视频比例归一化；像素尺寸转换为现有支持的标准比例。
- `duration` 将 `videoSeconds` 归一化为正整数秒数。
- `quality` 第一阶段固定为文档确认的 `720p`，不发送未确认的清晰度值。
- `image_urls` 保持有效公网参考图的原始顺序；文生视频使用空数组。
- `oversea` 第一阶段固定为 `false`。

文档未声明参考图数量上限，因此第一阶段不自行添加数量限制，也不静默截断。

### 6.2 查询任务

1. 每 2.5 秒请求 `GET /v1/videos/tasks/{task_id}`。
2. `state === "succeeded"` 时，从 `data.videos[].url` 提取所有有效 URL。
3. Infinite Canvas 当前一次视频生成返回一个结果，因此使用第一个有效视频 URL。
4. 服务端明确返回失败状态或错误字段时终止轮询，并显示服务端错误。
5. 尚未完成且没有明确失败时继续轮询。
6. 最多轮询 120 次，总等待时间约 5 分钟。
7. 成功状态没有有效视频 URL 时返回协议错误，并在错误信息中保留任务 ID。

任务 ID 必须进行 URL 编码后才能拼入查询路径。

### 6.3 结果存储

成功 URL 继续交给现有 `storeGeneratedVideo` 流程。能够下载时存入现有媒体存储；跨域或下载失败时保留远程 URL 和视频 MIME 类型，维持当前降级行为。

## 7. 公网参考图规则

多米参考图只接受可由中转站访问的公网 HTTP/HTTPS URL。

请求前拒绝：

- `data:` 和 `blob:` URL。
- 浏览器本地文件或仅存在于 IndexedDB 的图片。
- `localhost` 和 `.localhost` 主机。
- IPv4/IPv6 回环地址。
- IPv4 私网、链路本地和保留地址。
- IPv6 唯一本地、链路本地和未指定地址。
- 主机名为空、协议不合法或无法解析为 URL 的值。

校验失败时显示“多米 Grok 参考图仅支持公网图片 URL”及具体原因，并保证创建请求尚未发出。

第一阶段只做静态地址校验，不代表客户端能够证明远程图片长期可访问。远程 403、过期签名、防盗链或下载失败由多米任务错误返回。

## 8. 错误处理

- HTTP 401/403：提示检查 API Key、余额或模型权限，并附带服务端消息。
- HTTP 429：提示限流或额度不足，并附带服务端消息。
- 创建响应缺少任务 ID：显示创建响应协议错误。
- 成功响应缺少视频 URL：显示结果协议错误并保留任务 ID。
- 明确失败状态或错误字段：优先显示服务端错误消息。
- 未完成任务达到 5 分钟：显示“多米视频生成超时，请稍后重试”。
- 用户取消：抛出标准 `AbortError`，不继续轮询，不显示普通失败提示。
- 未知但无错误的中间状态：在超时范围内继续轮询，避免把供应商新增的处理中状态误判为失败。
- 网络或代理错误：沿用现有 Axios 错误读取逻辑，并使用多米视频上下文描述。

直连和现有 `/api/ai-proxy` 两种模式都必须保留。代理模式继续通过目标 Base URL 请求头指向渠道地址，但多米上游鉴权仍使用原始 API Key。

## 9. 测试策略

### 9.1 纯协议测试

使用现有 `node:test` 模式覆盖：

- 创建和查询路径。
- 任务 ID URL 编码。
- 原始鉴权头，不出现 `Bearer`。
- 完整请求体和文生视频空 `image_urls`。
- 公网参考图顺序保留。
- 比例、时长和 `720p` 映射。
- 顶层 `id` 提取。
- `succeeded` 状态和 `data.videos[].url` 提取。
- 服务端错误、空结果和异常结构。
- 公网、Base64、Blob、localhost、私网、回环和链路本地地址校验。

### 9.2 生命周期测试

使用模拟网络服务覆盖：

- 创建后轮询到成功。
- 服务端明确失败。
- 创建响应缺少任务 ID。
- 成功响应缺少视频 URL。
- 未知中间状态继续轮询。
- 超时停止。
- 创建阶段取消。
- 轮询阶段取消。
- 非公网参考图在联网前被拒绝。
- 直连和代理模式的 URL 与请求头。

### 9.3 配置与回归测试

覆盖：

- 旧渠道迁移为 `videoApiFormat: "standard"`。
- 多米视频协议加入 `grok-video-1.5` 建议模型。
- 多米图片和视频建议模型正确合并且去重。
- 切换协议后用户模型不丢失。
- 选择多米视频模型后按渠道协议分发。
- 现有 OpenAI、xAI、Seedance 和多米图片测试继续通过。
- TypeScript 检查、代理测试和生产构建通过。

### 9.4 浏览器验收

在隔离开发服务中验证：

- 渠道界面显示“视频协议”。
- 可以独立选择“多米异步视频”。
- “新增渠道”菜单可以分别创建空白渠道和多米 API 预设渠道。
- 多米渠道同时显示正确的图片和视频建议模型。
- `grok-video-1.5` 可以被选为默认视频模型。
- 公网参考图可以进入多米视频创建流程。
- 本地或私网参考图显示前置错误，且没有发出多米创建请求。
- 切回标准协议后渠道模型仍保留。

默认验收使用模拟服务，不消耗真实 API 额度。只有用户提供有效密钥并明确同意后才执行真实付费请求。

## 11. 第一阶段验收标准

- 旧配置迁移后行为不变，现有视频和图片协议不回归。
- 用户可以在渠道中独立选择多米图片和多米视频协议。
- 多米渠道模板包含已确认的图片模型和 `grok-video-1.5`。
- 文生视频请求使用多米文档规定的路径、鉴权和请求字段。
- 公网图片 URL 可以作为 `image_urls` 进入参考图视频请求。
- 本地、Base64、Blob 和非公网参考图在联网前被拒绝。
- 创建任务可以轮询到 `data.videos[].url` 并进入现有视频存储与画布流程。
- 取消、失败、超时、缺少任务 ID 和缺少视频 URL 均有明确处理。
- 模拟服务测试、现有回归测试、TypeScript 和生产构建通过。

## 12. 后续阶段

只有在用户明确需要本地参考图直连多米视频后，才单独设计图片上传能力。后续设计需要重新确认上传目的地、访问控制、生命周期、隐私、成本、清理策略以及第三方数据处理边界，不纳入本次接入。
