先看结论

它适合把“一句话”变成一组可复用素材

把 Infinite Canvas 想成一张不会用完的工作桌：提示词是便签，图片和视频是素材卡，线条表示它们之间的关系。

适合做什么

  • 边想边摆放图片、视频和文字
  • 用图片模型生成单图或参考图新视角
  • 用 480p / 4 秒小样验证视频创意
  • 把满意成果下载或收入“我的资产”
  • 把画布与媒体文件同步到 NAS WebDAV

今天没有承诺什么

  • 界面选择“4K”不等于接口返回真 4K
  • 本次没有把长视频、高分辨率视频作为稳定测试线
  • 没有把文本模型、音频模型列入今天的验收闭环
  • API Key 仍保存在本机浏览器，不会跟随 WebDAV 同步
6 个词先认脸

像整理桌面一样理解画布

画布

无限大的工作桌。你可以缩放、拖动并摆放多个任务。

节点

桌上的一张卡片。文字、图片、视频都是不同节点。

连线

素材之间的“传送带”，告诉系统谁给谁提供输入。

渠道

连接模型服务的账号与地址，像选择哪家快递公司。

模型

真正干活的 AI。图片与视频要选各自对应的模型。

资产 / 同步

资产是成果仓库；同步是把仓库副本保存到 NAS。

10 分钟最小闭环

第一次只做一张图

先确认整条生产线通了，再做参考图、视频或复杂工作流。这样出错时最容易找到是哪一站的问题。

启动本机服务

双击项目里的 start-local.command。终端窗口要保持打开，然后浏览器访问 http://localhost:3000。

成功信号能看到顶部“无限画布 / 生图工作台 / 视频创作台 / 配置”等导航。
没打开怎么办先看终端最后一行错误；不要连续重复双击多个启动窗口。

确认图片渠道

进入“配置 → 渠道”，确认图片渠道已存在，模型为 gpt-image-2。如果今天已经配置成功，不需要再次填写密钥。

成功信号模型列表里能选择 gpt-image-2。
读取模型失败不要反复点拉取；可手动增加准确的模型名称再保存。

生成第一张图片

打开“生图工作台”，输入一句清楚的提示词，例如“阳光下坐在木桌旁的一只橙色小猫”。先选 1 张、常规比例，再点“开始生成”。

成功信号左侧记录显示“成功 1”，右侧出现可预览结果。
先别做复杂设置首次不要同时开参考图、透明背景、多张和 4K。

保存并确认原图

在结果卡片点“下载”保存原图；需要继续创作时点“添加到…”或“加入参考”。不要只看页面里的预览尺寸。

成功信号下载目录中出现可正常打开的 PNG/JPG 文件。
关于 4K下载后在 Finder“显示简介”查看真实像素；界面选中 4K 只是请求参数，不是结果保证。
图片工作台

文生图与参考图怎么用

今天的图片链路已经实测成功。最稳的原则是：先单图、再加参考；先看真实文件、再谈分辨率。

真实成功截图 模型：gpt-image-2 数量：1 张 当次耗时：约 33 秒

文生图

  1. 输入“主体 + 场景 + 动作 + 构图”。
  2. 选比例、质量和张数。
  3. 点“开始生成”，等结果卡片出现。
  4. 下载原图或加入资产。

参考图生图

  1. 先把原图放进画布或“我的资产”。
  2. 在生成节点里添加为“参考图”。
  3. 提示词明确写“保持主体、颜色与材质，只改变视角/动作”。
  4. 如果返回 400，先确认参考图已上传完成，再重试。
分辨率验收规则 页面显示的宽高比、2K/4K 按钮是“你向模型提出的要求”；下载文件的实际像素才是“模型交付的货”。两者不一致时，以原文件为准。
视频创作台

先用 480p / 4 秒做便宜的小样

视频像打样：先花最少成本确认动作、画面和接口都正确，再决定是否制作更长、更高质量的版本。

我的画布

把一次生成变成可复用流程

放入文字节点

双击空白处或使用添加入口，创建文字节点，写下你的提示词。节点标题尽量用“用途 + 版本”，例如“橙猫主视觉 V1”。

连接生成节点

从文字节点的连接点拖线到生成配置节点。连线就像传送带：上游内容会成为下游生成的输入。

选模型与参数

图片选择 gpt-image-2；视频先选择 Seedance，并坚持 480p / 4 秒测试线。

检查、命名、保存

生成完成后先打开“信息”查看状态与错误详情，再把满意结果收入资产。一个画布只放同一主题，后续更容易找。

WebDAV 同步

把项目副本放进 NAS

WebDAV 像一根连接浏览器与 NAS 的网线。它同步画布、资产、生成记录和媒体文件，但不会同步 AI API Key。

输入项填写内容注意
WebDAV 地址http://192.168.1.44:5005本机要和 NAS 在可访问的网络内。
远程目录home/infinite-canvas保持固定，避免多个目录分散。
用户名填写 NAS 用户名只保存在当前浏览器本地。
密码 / 应用密码填写 NAS 密码不要写进代码、截图或 Git。

先点“测试连接”

只有出现连接成功，才说明地址、账户和浏览器代理链路都能访问。

再点“立即同步”

同步后在 NAS 对应目录应看到业务目录、manifest.json 和 files/。测试连接成功不等于文件已经同步。

遇到问题先看这里

报错不是终点，它是在告诉你该查哪一站

页面打不开或显示连接失败

确认启动终端仍然打开，再访问 http://localhost:3000。如果 3000 端口被占用，先关闭重复启动的服务窗口，再只启动一次。

读取模型失败 / 模型列表为空

部分中转不提供标准模型列表接口。进入“选择模型”，手动输入准确模型 ID 后添加并保存，不需要一直点击“拉取模型列表”。

图片请求返回 400

先用纯文字、1 张、普通比例测试。如果纯文字成功而参考图失败，再检查参考图是否上传完成、是否使用对应的图片编辑能力。一次只增加一个变量。

Seedance 任务创建失败

退回已验收组合：Seedance 2.0 Mini、480p、4 秒、关闭音频、无参考图。仍失败时打开节点“信息 → JSON”，保留 errorDetails 作为排查证据。

选择了 4K，下载后像素仍不够

这是上游模型或中转服务没有按请求返回目标像素，不是浏览器预览缩小造成的。以下载原图的像素、文件大小和服务使用记录为准。

WebDAV 测试成功，但 NAS 没看到文件

“测试连接”只证明能通信；还要点击“立即同步”，并检查远程目录下是否出现 manifest.json 与 files/。

菜鸟编程术语表

看得懂，就不怕排错

词汇作用生活类比与注意
本机服务让你的电脑运行 Infinite Canvas 网页。像在家里开一家只服务自己的小店；终端关掉，小店就打烊。
localhost指向当前这台电脑。像写给快递员的“送到我家”；别人电脑访问同一个地址不会进到你的电脑。
API网页向模型服务提交任务的接口。像餐厅点单窗口：你提交菜名和要求，后厨返回成品或报错。
API Key证明你有权调用模型并承担费用。像银行卡密码；不要放进代码、Git 或公开截图。
模型 ID指定由哪一个 AI 模型完成任务。像点单时写清厨师名字；少一个字符都可能找错人。
HTTP 400服务认为请求参数不符合要求。像订单格式不对被退回；先减少参数，一项一项加回去。
WebDAV让浏览器按文件方式读写 NAS。像一扇远程仓库门；能开门不代表货已经搬进去，还要执行同步。
Git记录代码与文档的版本变化。像项目的时间机器；密钥和密码不能被它记录。
