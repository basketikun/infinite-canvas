# 本地开发

主应用位于 `web/`，同时包含页面、Token OAuth 回调、加密会话和同源 AI 代理。

## 1. 准备环境变量

在 `web/.env.local` 中至少配置：

```dotenv
CANVAS_PUBLIC_ORIGIN=http://localhost:3000
TOKEN_ISSUER_URL=http://localhost:3001
TOKEN_API_BASE_URL=http://localhost:3001
CANVAS_OAUTH_CLIENT_ID=canvas
CANVAS_OAUTH_CLIENT_SECRET=replace_with_the_same_secret_as_token
CANVAS_OAUTH_REDIRECT_URI=http://localhost:3000/auth/callback
CANVAS_ENCRYPTION_KEY=replace_with_32_byte_base64_or_64_hex_key
CANVAS_DATA_DIR=.data
```

Token/New API 端必须使用相同的 Client ID、Client Secret 和精确回调地址，并启用 `image` 分组。

## 2. 启动前端

```bash
cd web
bun install
bun run dev
```

访问 `http://localhost:3000`，通过 Token 账号完成登录。真实 Token API Key 只保存在本地 `.data/canvas.db` 的加密字段中。

## 3. 启动文档站

```bash
cd docs
bun run dev
```

## 常见改动位置

- 登录、会话和 AI 代理：`web/src/app/api/auth/`、`web/src/app/auth/`、`web/src/app/api/ai/`、`web/src/lib/server/`
- 画布、页面和交互：`web/src/app/(user)/`
- 提示词缓存或 WebDAV 代理：`web/src/app/api/prompts/`、`web/src/app/webdav-proxy/`
- 文档站内容：`docs/content/docs/`
