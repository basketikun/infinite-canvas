# 快速开始

当前 Token 一键登录架构需要持久化的 Next.js 服务端会话，推荐使用 Docker 部署。无持久化 SQLite 数据盘的纯静态或 Serverless 部署不适合直接使用这套登录方式。

## Docker 启动

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
cp .env.example .env
# 编辑 .env，并为 CANVAS_ENCRYPTION_KEY 填入 `openssl rand -base64 32` 的结果
docker compose up -d
```

启动后访问：

```text
http://localhost:3000
```

生产环境还需要在 Token/New API 服务端配置相同的 `CANVAS_OAUTH_CLIENT_ID`、`CANVAS_OAUTH_CLIENT_SECRET` 和精确匹配的 `CANVAS_OAUTH_REDIRECT_URI`。

## 本地启动

```bash
cd web
bun install
bun run dev
```

本地环境必须把 `CANVAS_PUBLIC_ORIGIN` 和 OAuth 回调地址改为本地地址，并让本地 Token/New API 使用完全相同的回调配置。

## 首次使用

- 未登录时点击“使用 Token 账号一键登录”。
- Token 登录后会自动创建或复用 `image` 分组令牌并返回 Canvas。
- Canvas 会自动从 Token 更新可用模型；也可以在配置弹窗中手动刷新模型列表。
- 画布、素材和生成记录主要保存在浏览器本地；跨设备同步可自行配置 WebDAV。

## 安全说明

- 真实 Token API Key 不进入浏览器，也不会出现在 URL 中。
- API Key 以 AES-256-GCM 密文保存在服务端 SQLite 会话数据库中。
- 浏览器只请求同源 `/api/ai`，代理上游固定为 Token/New API，用户不能指定任意 Base URL。
