# Docker 部署

当前镜像运行 Next.js 页面、OAuth 回调、加密会话和同源 AI 代理。部署时必须配置环境变量并持久化 `/app/data`。

## 1. 准备配置

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
cp .env.example .env
openssl rand -base64 32
```

把随机结果写入 `.env` 的 `CANVAS_ENCRYPTION_KEY`，并填写与 Token/New API 服务端一致的 OAuth Client Secret。不要提交真实 `.env`。

Token/New API 端需要对应配置：

```dotenv
CANVAS_OAUTH_CLIENT_ID=canvas
CANVAS_OAUTH_CLIENT_SECRET=replace_with_the_same_long_random_secret
CANVAS_OAUTH_REDIRECT_URI=https://canvas.example.com/auth/callback
CANVAS_OAUTH_TOKEN_GROUP=Image
```

## 2. 启动

```bash
docker compose up -d
```

`docker-compose.yml` 默认使用 `ghcr.io/carminback/infinite-canvas:latest`，也可以通过 `CANVAS_IMAGE` 指定已验证的 SHA 镜像标签。生产部署建议固定到具体 `sha-...` 标签。

## 3. 数据卷

Compose 会把本地 `./data` 挂载到容器 `/app/data`。其中 `canvas.db` 保存：

- Session ID 的 SHA-256 哈希；
- Token 用户 ID 和用户名；
- 经 AES-256-GCM 加密的 Token API Key；
- 会话过期时间。

真实 API Key 不会写入浏览器存储或 URL。丢失 `CANVAS_ENCRYPTION_KEY` 后旧会话无法解密，需要用户重新登录。

画布、我的素材和生成记录仍主要保存在浏览器本地；WebDAV 同步需要用户自行配置。
