### 2. API 错误处理不统一

**问题描述**：错误处理风格不一致，部分返回 `Error` 对象，部分返回错误字符串

**代码示例**（`src/services/api/image.ts`）：
```typescript
catch (error) {
    throw new Error(readAxiosError(error, "请求失败"));
}
```

**问题**：
- `requestGeneration` 返回 `{ id, dataUrl }[]`
- 错误时抛出 `Error` 对象
- 建议统一为 `Result<T, Error>` 类型或使用 neverthrow

**建议改进**：
```typescript
// 方案 1：使用 Result 类型
type Result<T> = { ok: true; value: T } | { ok: false; error: string };

// 方案 2：使用 neverthrow
import { Result } from 'neverthrow';
async function requestGeneration(...): Promise<Result<ImageResult[], Error>>
```

### 2. 状态持久化优化

**当前实现**（`src/app/(user)/canvas/stores/use-canvas-store.ts`）：

setItem: (name, value) => {

// 批量请求去重
import { AbortController } from './abort-controller';

class RequestDeduplicator {
  private pending = new Map<string, AbortController>();

  async fetch(key: string, fn: () => Promise<any>) {
    if (this.pending.has(key)) {
      return this.pending.get(key)!;
    }
    const controller = new AbortController();
    this.pending.set(key, controller);
    try {
      return await fn();
    } finally {
      this.pending.delete(key);
    }
  }
}

### 3. 输入验证

**问题描述**：多处直接使用用户输入而未验证

**示例**（`src/services/api/image.ts:146-154`）：
```typescript
function parseImageRatio(value: string) {
    const parts = value.split(":");
    if (parts.length !== 2) throw new Error("图像尺寸格式不支持");
    const w = Number(parts[0]);
    const h = Number(parts[1]);
    // 缺少 NaN 检查
    if (w <= 0 || h <= 0) throw new Error("图像比例必须是正数");
}
```

**建议**：使用 Zod 进行 schema 验证

```typescript
import { z } from 'zod';

const ImageRatioSchema = z.string()
  .transform(val => val.split(':'))
  .refine(parts => parts.length === 2, '格式必须是 x:y')
  .transform(parts => ({
    width: Number(parts[0]),
    height: Number(parts[1])
  }))
  .refine(
    ({ width, height }) => width > 0 && height > 0,
    '宽高必须是正数'
  );

function parseImageRatio(value: string) {
  return ImageRatioSchema.parse(value);
}
```
